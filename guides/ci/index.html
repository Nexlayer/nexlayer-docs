    üîÑ Continuous Integration with Nexlayer

    You can deploy to Nexlayer from any CI pipeline using a simple curl call. There‚Äôs no special format required‚Äîjust build your image, push it to a registry, and post your nexlayer.yaml to the deployment endpoint.

    üí° Any technology that can be containerized can be deployed on Nexlayer ‚Äî from Node.js, Python, and Go, to ML workloads, microservices, or internal tools.

    ‚úÖ Basic CI Flow

    curl -X POST https://app.nexlayer.io/startUserDeployment \
  -H "Content-Type: text/x-yaml" \
  --data-binary @nexlayer.yaml

    If successful, the response will include a live URL:

    {
  "url": "https://yourapp-name.nexlayer.app"
}

    üß† What Your CI Should Do

    1. üê≥ Build your Docker image
    2. ‚òÅÔ∏è Push it to a public or private registry (recommended: GitHub Container Registry)
    3. üìÑ Update your nexlayer.yaml with the image reference (e.g., ghcr.io/your-org/your-app:tag)
    4. üöÄ Deploy using POST /startUserDeployment

    üìù Note on databases: You can connect to external managed databases (like MongoDB Atlas, Supabase, or Neon). While it works, we recommend deploying your own DB pod on Nexlayer for tighter control, lower latency, and environment isolation.

    üõ† GitHub Actions Example (with GHCR)

    name: Nexlayer CI/CD

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write

    steps:
      - uses: actions/checkout@v3

      - name: Log in to GitHub Container Registry
        run: echo "${{ secrets.GITHUB_TOKEN }}" | docker login ghcr.io -u ${{ github.actor }} --password-stdin

      - name: Build and push Docker image
        run: |
          docker build -t ghcr.io/${{ github.repository }}:latest .
          docker push ghcr.io/${{ github.repository }}:latest

      - name: Patch nexlayer.yaml with image
        run: |
          sed -i "s|image:.*|image: ghcr.io/${{ github.repository }}:latest|" nexlayer.yaml

      - name: Deploy to Nexlayer
        run: |
          curl -X POST https://app.nexlayer.io/startUserDeployment \
            -H "Content-Type: text/x-yaml" \
            --data-binary @nexlayer.yaml

    üí° Tip: GitHub‚Äôs native container registry (GHCR) is a secure and reliable choice for CI/CD deployments.

    üîî CI Notifications (Optional)

    To capture deployment status, capture the response:

    RESPONSE=$(curl -s -X POST https://app.nexlayer.io/startUserDeployment \
  -H "Content-Type: text/x-yaml" \
  --data-binary @nexlayer.yaml)

URL=$(echo "$RESPONSE" | jq -r .url)
echo "‚úÖ Deployed to: $URL"

    You can then send it to Slack or Discord using a webhook.

    üåê Endpoints Summary

    Action Endpoint
    Deploy POST https://app.nexlayer.io/startUserDeployment
    (Optional) GET https://app.nexlayer.io/schema ‚Äì generate YAML
    (Optional) POST https://app.nexlayer.io/validate ‚Äì validate YAML

    üì¶ Notes on Image Hosting

    • All images must be hosted on a registry (e.g., Docker Hub, GHCR)
    • Local images will not work‚ÄîNexlayer must be able to pull the image remotely

    üß™ Common Gotchas

    • ‚ùå Missing image: tag or incorrect reference in your YAML
    • ‚ùå Using a local-only Docker image not pushed to a registry
    • ‚ùå Not updating your nexlayer.yaml with the image tag you just pushed

    ‚úÖ Recap: Full Flow in CI

    1. Build your Docker image
    2. Push it to GHCR or another registry
    3. Update your nexlayer.yaml
    4. curl -X POST /startUserDeployment
    5. Parse and post the returned URL if needed

    This CI flow works across all platforms‚ÄîGitHub Actions, GitLab CI, CircleCI, etc.‚Äîbecause it‚Äôs just HTTP. You bring the image, Nexlayer brings the launchpad.
