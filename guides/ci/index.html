    Continuous Integration Guide

    This guide explains how to set up and configure continuous integration (CI) for your Nexlayer projects.

    Overview

    Nexlayer CI provides automated testing, building, and deployment capabilities for your applications. This guide will show you how to integrate Nexlayer deployments into your CI/CD pipeline.

    Setting Up CI

    Basic Configuration

    Create a .nexlayer/ci.yaml file in your project root:

    version: 1
stages:
  - name: test
    commands:
      - go test ./...
  - name: build
    commands:
      - go build -o app
  - name: deploy
    commands:
      - nexlayer deploy

    Advanced Configuration

    version: 1
stages:
  - name: test
    commands:
      - go test ./...
    environment:
      GO_VERSION: 1.16
      NODE_VERSION: 14
    timeout: 300
    retries: 3

  - name: build
    commands:
      - go build -o app
    artifacts:
      - app
    cache:
      - .go
      - node_modules

  - name: deploy
    commands:
      - nexlayer deploy
    conditions:
      - branch == main
      - test.success == true
    notifications:
      - type: slack
        channel: deployments

    GitHub Actions Integration

    Basic Workflow

    Create .github/workflows/nexlayer.yml:

    name: Nexlayer CI/CD

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Set up Go
        uses: actions/setup-go@v2
        with:
          go-version: 1.16

      - name: Install Nexlayer CLI
        run: |
          curl -fsSL https://get.nexlayer.io | sh

      - name: Run Tests
        run: go test ./...

      - name: Build
        run: go build -o app

      - name: Deploy to Nexlayer
        if: github.ref == 'refs/heads/main'
        run: |
          curl -X POST https://app.nexlayer.io/startUserDeployment \
            -H "Content-Type: text/x-yaml" \
            --data-binary @nexlayer.yaml

    Advanced Workflow with Caching

    name: Nexlayer CI/CD

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Set up Go
        uses: actions/setup-go@v2
        with:
          go-version: 1.16
          cache: true
          cache-dependency-path: go.sum

      - name: Install Nexlayer CLI
        run: |
          curl -fsSL https://get.nexlayer.io | sh

      - name: Run Tests
        run: go test ./...

      - name: Build
        run: go build -o app

      - name: Cache Build Artifacts
        uses: actions/cache@v2
        with:
          path: app
          key: ${{ runner.os }}-build-${{ hashFiles('**/go.sum') }}
          restore-keys: |
            ${{ runner.os }}-build-

      - name: Deploy to Nexlayer
        if: github.ref == 'refs/heads/main'
        run: |
          DEPLOY_URL=$(curl -X POST https://app.nexlayer.io/startUserDeployment \
            -H "Content-Type: text/x-yaml" \
            --data-binary @nexlayer.yaml | jq -r '.url')
          echo "Deployed to: $DEPLOY_URL"

      - name: Notify Deployment
        if: success()
        uses: rtCamp/action-slack-notify@v2
        env:
          SLACK_WEBHOOK: ${{ secrets.SLACK_WEBHOOK }}
          SLACK_MESSAGE: "Deployment successful!"
          SLACK_COLOR: good

    Deployment Flow

    The Nexlayer deployment process follows these steps:

    1. Schema Validation (GET /schema)

    2. Validates your YAML against the Nexlayer schema

    3. Ensures all required fields are present and correctly formatted

    4. Configuration Validation (POST /validate)

    5. Validates your application configuration

    6. Checks for potential issues or conflicts

    7. Deployment (POST /startUserDeployment)

    8. Deploys your application to Nexlayer
    9. Returns a deployment URL

    Example Deployment YAML

    Here's an example nexlayer.yaml file for a typical web application:

    application:
  name: "my-web-app"
  url: "www.example.ai" # Optional: Include for permanent deployments
  pods:
    - name: frontend
      image: "your-username/frontend:v1.0.0"
      path: "/"
      servicePorts:
        - 3000
      vars:
        API_URL: "http://backend.pod:4000"

    - name: backend
      image: "your-username/backend:v1.0.0"
      path: "/api"
      servicePorts:
        - 4000
      vars:
        DATABASE_URL: "postgresql://postgres:password@database.pod:5432/mydb"

    - name: database
      image: "postgres:14"
      servicePorts:
        - 5432
      vars:
        POSTGRES_USER: "postgres"
        POSTGRES_PASSWORD: "password"
        POSTGRES_DB: "mydb"
      volumes:
        - name: db-data
          size: "1Gi"
          mountPath: "/var/lib/postgresql/data"

    Environment Variables

    Required Variables

    NEXLAYER_API_KEY: Your API key
NEXLAYER_ENV: production

    Optional Variables

    NEXLAYER_REGION: us-west-2
NEXLAYER_TIMEOUT: 300
NEXLAYER_RETRIES: 3

    Artifacts and Caching

    Artifact Configuration

    artifacts:
  - path: app
    retention: 7d
  - path: coverage.out
    retention: 30d

    Cache Configuration

    cache:
  - path: .go
    key: go-mod-${{ hashFiles('go.sum') }}
  - path: node_modules
    key: node-mod-${{ hashFiles('package-lock.json') }}

    Notifications

    Slack Integration

    notifications:
  - type: slack
    channel: deployments
    events:
      - deploy.success
      - deploy.failure

    Email Notifications

    notifications:
  - type: email
    recipients:
      - team@example.com
    events:
      - deploy.success
      - deploy.failure

    Security

    Secrets Management

    secrets:
  - name: NEXLAYER_API_KEY
    value: ${NEXLAYER_API_KEY}
  - name: SLACK_WEBHOOK
    value: ${SLACK_WEBHOOK}

    Access Control

    access:
  roles:
    - name: admin
      permissions: [all]
    - name: operator
      permissions: [read, execute]
